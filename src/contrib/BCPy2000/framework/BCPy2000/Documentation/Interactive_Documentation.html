Interactive Documentation

Use the shell to explore. Two of IPython's most useful aides to climbing any learning curve are the question-mark and the tab key.

For example, in the shell for a running BCPy2000 module, type self. and then, without pressing return, press tab. Tab-completion allows you to list the attributes of any object easily (as well as saving on typing).

Now, complete the expression by typing/tabbing until you get, say, self.debug . Now append a ? and press return. The documentation for that method appears. Much of BCPy2000 documentation exists in this online form, so it is worth getting accustomed to this way of working. Even when no natural-language documentation exists for a given function, IPython prints the prototype, usually showing you the argument names and their defaults, which can be a good start.

Next, compare the effect of a single-questionmark 

self.debug?

with that of a double-questionmark 

self.debug??

The latter gets you not only the documentation, but the source code for that method.

Typing ? alone gets you an introduction to yet more of IPython's useful features.

Interactive BCPy2000 Documentation

Try this: 

from BCPy2000.Documentation import *

This gives you the symbols core, app, sig and src which are in fact just aliases to the BciCore, BciGenericApplication, BciGenericSignalProcessing and BciGenericSource classes. This makes it easy to browse the docstrings: for example, you can easily see that 

src.Process?

gives you somewhat different (more source-module-specific) documentation than 

core.Process?

Finally, the BCPy2000.Documentation module also gives you the browse() function, which opens these html pages (assuming your desktop environment is still set up such that html files open in a browser rather than an editor when double-clicked).
