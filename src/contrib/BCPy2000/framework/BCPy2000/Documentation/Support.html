Jez says:

To contact me, you might want to concatenate the name jez, that funny curly "a" symbol, and the domain tuebingen.mpg.de, and see what you get.

If you're going to ask for support, please think first about how you can supply the right level of detail for me to be able to identify the context in which the problem occurs (and, ideally, to re-create it). I can't see your screen very well from here.

Familiarize yourself with the types of errors that might occur, and learn to interpret the various cues that tell you where they are coming from. In your support request, quote any error backtraces that you get.

Another useful piece of information is the output you get from importing the BCPy2000.Diagnostics module. Among other things, this lists the versions of the various dependencies you have installed.

Note also that there may already be a few known issues with this release, which you should check.

Finally, note that this is the place for support for the python-related BCI2000 components. If your problem doesn't originate in a component that has Python or Py or .py in its name, then it may be an issue for BCI2000 support instead.
