Changelog

2009-03-16

First service-pack build. Version numbering of the BCPy2000 Python package now uses the revision numbers from the MPI subversion repository, where we manage our development copy. This means we have jumped from version 1.0 to version 11336, which is clearly quite some progress. The bci2000.org revision number corresponding to this release is 2324.

  • BCPy2000 modules are now all gnu-compilable and can be made to run on non-Windows systems.
  • In the application module, the VisionEgg main loop has been rearranged and streamlined, in the continuing quest for more reliable video timing.
  • Robuster parameter interpretation in BCI2000Tools/FileReader.py, and a fix in BCI2000Tools/PlaybackSourceModule.py fix errors that occurred when playing back some types of data file (reported by Marco Rotonda).
  • In the application module when running under Windows, two bits of automatic housekeeping are performed. On StartRun the new self.focus() method is used to raise the VisionEgg window, and the new self.volume() method is used to set the system volume to maximum. In StopRun. the same methods are called to re-raise the Operator window and set the system volume back to what it was at initialization time.
  • A bug-fix in WavTools makes the timing of sound stimuli more precise when played using the PyAudio interface. Also, panning is more flexible, and the default method of normalizing different channels' gains has changed, meaning that centre-panned sounds are louder by default than they used to be.
  • In SigTools, more flexible wavegen replaces sinewave.
  • Numerous small bug-fixes and enhancements.

2008-09-30

Initial release 1.0.
