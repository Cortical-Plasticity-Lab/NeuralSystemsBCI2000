SigTools

This package is part of the evolving set of optional higher-level tools. It consists of:

  • SigTools.Basic contains various general-purpose helper functions, including a function for generating sinewaves, and functions for splitting discrete Fourier transforms into amplitude and phase spectra while avoiding the fiddly edge conditions this often entails.
  • SigTools.Filtering contains various tools for processing signals online, including objects called causalfilter, fader and trap.
  • SigTools.LearningTools contains various tools such as a convenient wrapper for performing Singular Value Decompositions, an implementation of the Common Spatial Pattern algorithm and Linear Discriminant Analysis classifier, and objects for computing means and covariances stably online.
  • SigTools.NumTools defines various simple Matlab-like tools, including enhanced routines for reading and writing .mat files.
  • SigTools.Plotting defines some wrappers around pylab plotting routines, plot and plotsig (the latter for plotting a time-series and its amplitude spectrum simultaneously).
  • SigTools.Stimuli defines a couple of simple amplitude-modulated sinewaves for use as one-dimensional stimulus signals.

Future releases may include some example BciSignalProcessing implementations based on these tools. One very simple common BCI approach for classifying imagined movement might be sketched out as follows (for now, the exact implementation is left as an exercise for the reader):

  1. filter the incoming signal packet with a SigTools.Filtering.causalfilter object designed for the frequency band of interest;
  2. compute the (non-central) covariance matrix (don't subtract the mean, since this may be inaccurately estimated from one short packet: in any case, either your filtering has done that job already, or you're actually interested in the non-centrality, as in Guido Dornhege's CSP-for-SCP approach);
  3. use SigTools.LearningTools.running_mean objects (one per class) to accumulate per-class covariance estimates;
  4. at some point, use a SigTools.LearningTools.csp to compute spatial filters based on the running-means-of-matrices, then select a useful subset of these filters;
  5. spatially filter the signal, square the result, and smooth the result of that with another causalfilter object (low-pass this time) either before or after taking the log;
  6. use SigTools.LearningTools.running_cov objects (again, one per class) to obtain running estimates of the mean and covariance of the log bandpower features in each class;
  7. at some point, train a SigTools.LearningTools.lda object on these;
  8. ...

Spatial filters and LDA weight vectors can be written back into the BCI2000 system's parameters during the StopRun stage, provided you have requested write access during Preflight. Then they will be ready for use in the next run (and saved in the next .dat file).

Builders of ERP-based BCI systems might be interested in the SigTools.Filtering.trap object...
