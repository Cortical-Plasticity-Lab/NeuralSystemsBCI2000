Building

Building the foundation binaries

The .exe files of the foundation are built in .\demo\prog from C++ sources in .\foundation. Like all BCI2000 modules, these binaries can currently only be built using the Borland C++ compiler (they're working to change this). To build, run BuildDemo.bat, but make sure that Borland's bpr2mak.exe and make.exe are on the path, and not overshadowed by anything of the same name (e.g. GNU make under cygwin).

BuildDemo.bat does the following:

  1. builds (PythonSource.exe, PythonSignalProcessing.exe and PythonApplication.exe) in .\demo\prog\
  2. builds the operator from the core part of the BCI2000 source distribution (..\..\core\Operator\operat.bpr) and copies the result (..\..\..\prog\operat.exe) to .\demo\prog

Both steps require BCPy2000 to be in its correct location within the main BCI2000 source tree. If you do not have the full BCI2000 source (i.e. you have just downloaded the demo from another location) then the .exe files should be included already.

If you're not interested in the demo, BCI2000's main build script will also build the BCPy2000 foundation binariesfor you, along with everything else (actually it builds them first in .\demo\prog, and then copies them up to ..\..\..\prog from there). The .bat, .prm, .py and .wav files belonging to the demo are not included in the main BCI2000 tree unless you run Merge.bat

Installing the Python framework

First you will need to ensure that Python, and all the necessary third-party Python packages, are installed.

Then you'll need to install the BCPy2000 Python framework. If a BCPy2000-*.exe installer is already provided, you can run that. If not, then launching the InstallFramework.bat script will build an installer from the sources in .\framework and then launch it. Alternatively, you can run python .\framework\setup.py install by hand, but on Windows the binary installers have the advantage that they create and register uninstallers for you, which you can then access from the "Add and Remove Programs" control panel.

Testing

The demos are launched by double-clicking one of the demo batch files in the .\demo\batch directory.

If you have the full BCI2000 source distribution then, once the foundation binaries are built, you can choose to merge the demo into the main BCI2000 tree with Merge.bat and use them from there.
