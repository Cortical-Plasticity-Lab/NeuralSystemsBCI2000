Supplying a Parameter file and Startup Options

Several options can be passed to the Operator on startup, including the name of a BCI2000 parameter file, and the IP address for each module.

To supply a parameter file on startup, enter the name of a BCI2000 parameter file in the Parameter File box, either by typing the name in, or by pressing the [...] button and selecting the file. The parameters actually used will depend on which modules connect to the operator.

The IP addresses supplied are used to setup intermodule communications. Because BCI2000 modules communicate using the TCP/IP protocol, the Operator must know what the IP address for each module is to allow communication. If all of the modules are running on the same computer, then the default value of 127.0.0.1 is given to specify that the module is running on the local machine, and not from a different computer.
